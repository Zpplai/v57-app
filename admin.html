Administradores

Como definir um admin (manualmente):
1. No Firebase Console → Authentication, copie o UID do usuário.
2. Use o Cloud Shell ou Cloud Functions para rodar:

admin.auth().setCustomUserClaims(uid, { admin: true })
3. O usuário precisa fazer logout/login para o token atualizar.
(O cliente não pode alterar claims — requer Admin SDK no backend.)

📣 Anúncios (v51)

Sistema de anúncios com 3 provedores, modo de teste e painel F12 de debug. Tudo configurado neste navegador (localStorage).

🔧 Controles globais

🧩 Provedores (até 3)

O sistema tenta o provedor de prioridade 1 primeiro; se falhar (timeout, adblock), passa ao 2, depois ao 3. Cole a URL do invoke.js/script do provedor. O Container ID é exigido por alguns provedores (ex.: Adsterra).

🛠️ Manutenção do Sistema

Quando ativado, todos os usuários veem a tela: "Sistema em manutenção, tente novamente mais tarde."

📢 Aviso na entrada do site

Quando ativado, todos os usuários veem uma janela ao entrar no site com a mensagem que você escrever aqui. Cada vez que você salva, o aviso reaparece (mesmo que já tenha sido visto).

🧹 Forçar limpeza de cache

Quando ativo, ao tocar em "Forçar atualização agora", todos os usuários terão o cache do navegador limpo automaticamente e o app será recarregado na versão mais nova. Útil quando você sobe uma versão nova e quer garantir que ninguém fique numa versão antiga.

O botão vermelho aumenta a versão e dispara a limpeza imediata em todos os dispositivos conectados.
